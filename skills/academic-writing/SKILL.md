# 学术论文写作

系统化的学术论文写作流程，确保产出高质量、符合学术规范的论文。

## 何时使用

**必须使用：**
- 用户需要撰写学术论文（研究论文、综述论文）
- 用户需要进行文献调研和综述
- 用户需要准备学术投稿

**可选使用：**
- 需要学术性的技术文档
- 需要严谨的研究报告

## 学术写作五阶段流程

### 阶段 1：文献调研

**目标：** 全面了解研究领域，建立理论基础

**步骤：**
1. **明确研究问题**
   - 论文主题是什么？
   - 研究问题/假设是什么？
   - 目标期刊/会议是什么？
   - 预期贡献是什么？

2. **关键词提取**
   - 从研究问题中提取核心关键词
   - 考虑同义词和相关术语
   - 准备中英文关键词

3. **文献搜索**（使用 search_papers 工具）
   - 搜索近 3-5 年的高引用论文
   - 覆盖多个数据源（arXiv、Scholar、Semantic Scholar）
   - 每个关键词搜索 10-20 篇相关文献

4. **文献筛选**
   - 根据相关性排序
   - 优先选择高引用、近期、顶会/顶刊论文
   - 筛选出 10-15 篇核心文献

5. **深度阅读**（使用 fetch_paper 工具）
   - 获取核心文献的详细内容
   - 提取：研究问题、方法、结果、贡献
   - 整理：优点、局限、启发

6. **文献总结**
   - 按主题分类整理文献
   - 识别研究空白和机会
   - 明确本研究的定位

**输出：** 文献综述表格 + 研究定位说明

---

### 阶段 2：大纲设计

**目标：** 构建清晰的论文结构框架

**步骤：**
1. **确定论文类型和结构**

**研究论文结构：**
```
1. Abstract（摘要）
2. Introduction（引言）
   - 背景和动机
   - 研究问题
   - 主要贡献
   - 论文结构
3. Related Work（相关工作）
   - 按主题分类综述
   - 对比分析
   - 本研究的差异
4. Methodology（方法）
   - 问题定义
   - 方法设计
   - 技术细节
5. Experiments（实验）
   - 实验设置
   - 数据集
   - 评估指标
6. Results（结果）
   - 主要结果
   - 对比分析
   - 消融实验
7. Discussion（讨论）
   - 结果解释
   - 局限性
   - 未来工作
8. Conclusion（结论）
9. References（参考文献）
```

**综述论文结构：**
```
1. Abstract
2. Introduction
3. Background（背景知识）
4. Survey（分类综述）
   - 按方法/应用/时间分类
   - 每类详细分析
5. Comparative Analysis（对比分析）
6. Future Directions（未来方向）
7. Conclusion
8. References
```

2. **章节细化**
   - 为每个章节设计 2-4 个子章节
   - 列出每个子章节的核心要点
   - 标记需要的图表

3. **引用规划**
   - 标记每个章节需要引用的文献
   - 确保引用分布合理

4. **用户确认**
   - 展示完整大纲
   - 获取反馈并调整

**输出：** 详细的论文大纲（含子章节和要点）

---

### 阶段 3：内容撰写

**目标：** 按大纲撰写高质量的学术内容

**撰写原则：**
- **学术严谨**：所有观点有文献支撑
- **逻辑清晰**：章节间过渡自然
- **语言规范**：正式、客观、准确、简洁
- **引用规范**：使用标准引用格式

**撰写流程：**
1. **分章节撰写**
   - 按大纲顺序逐章节撰写
   - 每完成一个章节，使用 write_file 保存
   - 建议顺序：Method → Experiments → Results → Related Work → Introduction → Abstract

2. **引用插入**
   - 使用 citation_tool 生成标准引用
   - 确保引用格式一致（APA/IEEE/MLA）
   - 在文中正确标注引用位置

3. **图表说明**
   - 为每个图表撰写清晰的标题
   - 在正文中引用并解释图表
   - 确保图表编号连续

4. **过渡衔接**
   - 章节开头：说明本章内容和目的
   - 章节结尾：总结要点，引出下一章
   - 段落间：使用过渡词保持连贯

**输出：** 完整的论文初稿

---

### 阶段 4：质量检查

**目标：** 确保论文达到发表标准

**检查维度：**
1. **可读性（Readability）**
   - 句子长度适中（15-25 词）
   - 段落结构清晰（3-5 句）
   - 避免复杂嵌套句式

2. **学术性（Academic Tone）**
   - 使用正式学术语言
   - 避免口语化表达
   - 保持客观中立

3. **结构完整性（Structure）**
   - 所有必要章节齐全
   - 章节逻辑连贯
   - 篇幅分配合理

4. **引用规范（Citations）**
   - 所有引用格式正确
   - 引用分布合理
   - 参考文献完整

**检查流程：**
1. 使用 analysis_tool 分析论文质量
2. 根据分析结果识别问题部分
3. 逐项修复问题
4. 再次分析，迭代优化
5. 直到所有指标达标

**输出：** 质量报告 + 优化后的论文

---

### 阶段 5：格式规范

**目标：** 符合目标期刊/会议的投稿要求

**步骤：**
1. **格式转换**
   - 使用 convert_tool 转换为目标格式
   - 常见格式：LaTeX、Word、PDF

2. **模板应用**
   - 应用期刊/会议的官方模板
   - 调整页边距、字体、行距

3. **最终检查**
   - 检查格式是否符合要求
   - 确认字数限制
   - 验证图表质量

4. **交付**
   - 提供最终版本
   - 提供源文件（.tex 或 .docx）
   - 提供补充材料（如需要）

**输出：** 符合投稿要求的最终版本
