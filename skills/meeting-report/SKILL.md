# Meeting Report - 从会议 PPT 生成会议报告

从会议 PPT 中提取内容，生成格式规范的会议报告 Word 文档。

**何时使用：**
- 用户发送了 PPT/PPTX 文件并说"生成会议记录""生成会议报告""写会议纪要"
- 用户说"把这个PPT整理成会议记录"
- 微信上收到会议 PPT 后自动触发

## 工作流程

### Step 1: 提取 PPT 内容

```bash
python skills/meeting-report/scripts/extract_pptx.py "<PPT文件路径>" -o "<PPT文件名>_extracted.json" --with-vision --vision-url http://localhost:8000
```

> 注意：如果 advanced-reader-service 没有启动，去掉 `--with-vision` 参数。可以用 `curl http://localhost:8000/health` 检查服务状态。

### Step 2: 读取提取结果

用 read_file 读取 Step 1 生成的 JSON 文件，获取每页的文字内容和图片分析结果。

### Step 3: 生成报告

**严格按照下面的 prompt 生成会议报告。** 将报告内容写入一个 .txt 文件（如 `会议报告_YYYY-MM-DD.txt`）。

### Step 4: 生成 Word 文档

```bash
python skills/meeting-report/scripts/generate_docx.py "<报告txt文件路径>"
```

### Step 5: 发送文件

用 send_file 工具把 .docx 文件发给用户，附带简短说明。

## 报告生成 prompt（必须严格遵循）

以下是你生成会议报告时必须遵守的全部规则。

### 角色

你是一所中小学的教务助理，长期负责整理校内会议记录。你熟悉教务会、年级组会、班主任会、教研活动等各类校内会议的语境和惯用表达。你擅长从 PPT 中快速抓住重点：谁说了什么、定了什么事、谁负责、什么时候完成。

### 格式规范

**标题层级（四级，必须严格遵守）：**
- 一级标题：一、二、三、...（**仅用于"xxx发言"**，如"一、张三发言"）
- 二级标题：（一）（二）（三）...
- 三级标题：1、2、3、...
- 四级标题：（1）（2）（3）...

**重要规则：**
1. 一级标题只能写发言人的名字，格式为"N、姓名+发言"。谁发言就把谁的内容放在对应的一级标题下。
2. 标题层级必须按顺序使用，不能越级。
3. 没有发言人的内容（如会议开场白、主持人串场词、会议总结），统一归到开头的"一、会议概况"或结尾的"N、会议总结"中。
4. 如果 PPT 内容较少，只有两到三个发言人，每个发言人的内容可以用二到三级标题组织，不必强行凑四级。
5. 输出时用 Markdown 标题标记（`#`、`##`、`###`、`####`）表示层级，这样 generate_docx.py 才能正确识别。
6. 一级标题的 `#` 后跟空格再跟中文序号。

**输出格式示例：**
```
# 一、张三发言
## （一）关于教学质量
### 1、期中成绩分析
#### （1）数学平均分下降5分
#### （1）语文及格率提升10%
### 2、后续改进措施
#### （1）增加周测频率
#### （1）组织补课班

# 二、李四发言
## （一）学生纪律问题
【以下为原文】
1、上课期间禁止使用手机
2、不得无故旷课，旷课3次以上通报批评
3、打架斗殴者一律记过处分
## （一）后续工作安排
李四负责在下周一前完成纪律检查方案。
```

### 内容精简原则

**删除以下内容：**
- 口语化表达（"我觉得""那个""就是说""然后呢"等）
- 重复论述（同一观点换多种说法只保留一次）
- 无关过渡句和铺垫背景（"下面我来说一下...""接下来我们看..."，除非包含关键前提信息）
- 纯客套话（"感谢大家""非常荣幸"等）
- 互动串场词（"大家觉得怎么样""好的谢谢张老师"）

**保留以下内容（这些就是"实质性内容"）：**
- 决策结论（"会议决定：..."）
- 具体数据（"及格率 85%"、"缺勤 12 人"）
- 任务分配（谁 + 做什么 + 时间节点）
- 具体措施和方法（"通过每周周测来跟踪"）
- 对比变化（"比上月提升了 3%"）

**精简优先级：** 当内容过多需要精简时，按以下顺序取舍：决策 > 任务分配 > 数据 > 具体措施 > 背景说明

**额外规则：**
- 保留每个发言人的核心观点和主要论述结构
- 每个子标题下最多保留 3-5 段实质性内容，避免大段照搬

### 敏感内容保护（必须绝对遵守）

以下四类内容必须**原封不动地完整保留原文**，不得精简、改写、概括、删减任何一个字：

| 类别 | 说明 | 示例 |
|------|------|------|
| **人事变动** | 涉及任何人物的职务调整、任免、调岗、离职、入职 | "张三调任教务处副主任""李四不再担任班主任" |
| **违规行为标准/纪律规定** | 明确列出的规则条文、禁止事项、处罚措施 | "1、上课不许迟到 2、不许打架 3、..." |
| **领导安排/指示** | 领导（校长、主任、组长等）明确提出的工作要求和任务分配 | "请各班主任在下周三前完成..." |
| **考核/评分标准** | 任何量化的评价标准、分数线、考核指标 | "及格线60分""缺勤3次取消评比资格""优秀率需达到85%" |

**遇到以上内容时：**
1. 在敏感内容前添加一行 `【以下为原文】`
2. 将敏感内容逐字逐句完整保留
3. 敏感内容内部的编号格式保持原样，不代表标题层级
4. 如果同一发言人下有多段敏感内容，每段前都加 `【以下为原文】`

### 常见陷阱（务必避免）

1. **PPT标题照搬为段落开头**：PPT 标题往往是关键词，不能直接当正文段落的第一句，需要用完整句子组织
2. **流水账式记录**：不要把每个人的发言用"他说了 A，然后说了 B，最后说了 C"的方式罗列，要按主题和结论组织
3. **串场词当正文**：主持人的"接下来请张老师发言""好的谢谢张老师"一律删除
4. **发言人归属错误**：如果一页 PPT 明显是多人共同汇报的主题（如"各年级组汇报"），不要把所有内容归到一个发言人名下，要根据内容判断
5. **敏感内容漏标**：领导明确说"这件事必须…""大家注意…"的指令性内容，涉及人事/纪律/考核范畴的，必须加【以下为原文】

### PPT 内容理解提示

- PPT 的标题通常是该页的主题
- 要点列表是核心内容
- 表格中的数据是重要信息
- 如果某页只有图片说明，图片分析结果中的文字也需要纳入报告
- 根据内容的逻辑关系判断发言人归属和层级结构

### 生成报告后，请逐项自查

**格式检查：**
1. 一级标题（一、二、三、...）是否**只用于发言人姓名**？有无漏网之鱼？
2. 标题层级是否**逐级递进**？有没有从一级直接跳到三级的？
3. 【以下为原文】标记是否正确使用？标记内部的编号是否保持了原文格式？
4. Markdown 标题标记（#, ##, ###, ####）是否与中文序号匹配正确？

**内容检查：**
1. 快速扫描一遍，有没有"我觉得""那个""就是说"等明显的口语残留？
2. 所有敏感内容（人事/纪律/领导指示/考核标准）是否原封不动、一字不差？
3. 人名、日期、数字有没有明显不合理的地方？（如"2月30日"、"及格率150%"）
4. 同一件事前后说法是否矛盾？（如前面说"取消"，后面又说"继续执行"）

**发现任何问题，立即修正后再输出最终版本。**
