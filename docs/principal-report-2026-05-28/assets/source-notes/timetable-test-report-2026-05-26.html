🔴 问题清单

问题1: `--help` 被安全策略拦截 中优先级

现象: 执行 `--help` 返回错误信息，而不是显示帮助文档

$ skill_cli --help
{
  "ok": false,
  "message": "--help 不能通过老师端排课助手执行。",
  "status": "failed"
}

影响: 用户无法查看可用命令列表，影响可用性

建议: 在白名单中加入 `help` 命令，或在 SKILL.md 中提供命令参考文档

问题2: `rule add` 效果不明确 高优先级

现象: 执行规则添加后返回 `ignored_rules`，让用户困惑

命令: rule add "金婷老师周三下午有教研"
结果: "ignored_rules": ["金婷老师周三下午不排课..."]

影响: 用户不清楚规则是否被保存，不知道如何让规则生效

建议: 将 `ignored_rules` 改为 "暂存规则（重新生成课表后生效）"，并提示执行 `resolve-imported`

问题3: `resolve-imported` 后冲突数量增加 中优先级

现象: 导入后2个冲突 → 重新生成后4个冲突

问题: 刘柯辰在周四第8节同时带3个班（音乐+舞蹈+心理）

建议: 确认这是算法优化结果还是 bug，需要进一步验证

问题4: 初始状态显示已有数据 低优先级

现象: 刚部署的系统显示 `source_mode: "imported_schedule"`，有6个班级数据

建议: 添加提示 "当前显示的是上次保存的课表"

问题5: `data patch` 提示不够清晰 低优先级

现象: 返回 "当前课表来自资料包；为了避免写入后看不见，请先决定..."

建议: 改为用户友好描述，提供明确的二选一选项

问题6: `move` 默认不覆盖已有课程 低优先级

现象: 不加 `--force` 时失败

建议: 错误提示改为 "目标位置已有课程，如需覆盖请加 --force"

问题7: 截图记录方式 说明

现象: CLI 环境限制，截图以 txt 文件保存

说明: 本报告已用 HTML 可视化呈现关键结果

🔧 修复优先级

优先级 问题 修复方案
🔴 高 问题2: rule add 提示 将 `ignored_rules` 改为 "暂存规则" 并提示如何生效
🟡 中 问题1: --help 拦截 白名单加入 `help` 命令
🟡 中 问题3: 冲突增加 确认 resolve-imported 算法逻辑
🟢 低 问题4/5/6 体验优化，后续迭代
