# 排课系统维护说明

这个项目现在是本地 Web 演示版，主要由 FastAPI 后端和普通 HTML/CSS/JS 前端组成。

## 常用维护位置

- 页面结构：`app/static/index.html`
  - 改按钮、输入框、页面区块、文字位置。
- 页面样式：`app/static/styles.css`
  - 改颜色、宽度、三栏布局、课表格子大小、按钮样式。
- 页面交互：`app/static/app.js`
  - 改按钮点击、规则保存、上传预览、拖拽换课、导出触发。
- 后端接口：`app/main.py`
  - 改 `/api/solve`、导出、Excel 预览等接口。
- 演示数据：`app/data.py`
  - 改小学/初中班级、课程课时、教师姓名、教室容量、默认时间。
- 规则识别：`app/rules.py`
  - 改“老师自然语言怎么被识别成排课条件”。
- 排课算法：`app/solver.py`
  - 改教师冲突、教室冲突、同科分散、连堂、课程均匀分布等逻辑。
- 导出文件：`app/exporters.py`
  - 改 Excel、PDF、CSV 的导出样式和内容。
- 自动测试：`tests/`
  - 改完功能后用来验证没有把已有功能改坏。

## 运行方式

在 `D:\ai-timetable-demo` 下运行：

```powershell
.\.venv\Scripts\python.exe -m uvicorn app.main:app --host 127.0.0.1 --port 8008
```

浏览器打开：

```text
http://127.0.0.1:8008
```

## 为什么按钮会出现“没反应”

常见原因有三类：

1. 浏览器缓存了旧的 `app.js`，页面结构和脚本版本对不上。
2. 页面脚本启动时报错，后面的点击事件没有绑定。
3. 浏览器本机保存过旧数据，把课程数据保存成空，导致右侧课表全是“自习”。

当前已经做了处理：

- `index.html` 给 CSS/JS 加了版本号，减少旧缓存影响。
- 页面初始化失败会在“排课反馈”里显示错误原因。
- 本机保存的课程/教室数据如果不可用，会自动退回演示数据。

## 正式交付时建议

- 老师端不要让老师改代码，只通过网页维护教师、课程、教室、时间和规则。
- 正式数据不要只存在浏览器本机，应保存到后端数据库或本地数据文件。
- Excel 导入应从“预览”升级为“字段映射后正式导入”。
